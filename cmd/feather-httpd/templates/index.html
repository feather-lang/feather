Feather

A lightweight, embeddable TCL interpreter. Add an interactive console to any application.

Get Started View on GitHub
feather repl 
# Define a greeting procedure
proc greet {name} {
    return "Hello, $name!"
}

% greet Feather
Hello, Feather!

# Work with lists
set numbers {1 2 3 4 5}
foreach n $numbers {
    if {$n > 3} { break }
    echo $n
}

route GET / {response "Hello, World!"}

Why Feather?

A pure interpreter designed for embedding. No I/O, no event loop, no surprises.

⚙

Zero Dependencies

Pure implementation with no stdlib. All I/O and allocation controlled by your host program.

⚡

Lightweight

Small footprint, fast startup. Interpreter objects are cheap enough to create on demand.

🔍

Fully Introspectable

Every aspect of a running program can be inspected and modified from within.

🧩

TCL Semantics

Faithful implementation of TCL core: procs, control flow, lists, dicts, namespaces, and more.

💻

Host Integration

Expose your application's objects and functions directly to Feather scripts.

🤖

Agent-Friendly

Let AI agents interact with your running application through a scriptable interface.

Built For

Feather fills the niche where you need an interactive console for your running application.

  • ✓ Runtime configuration and hot-reloading
  • ✓ Quake-style debug consoles
  • ✓ Agent access to application internals
  • ✓ Networked REPLs for remote inspection
  • ✓ User-customizable automation
  • ✓ Declarative configuration files

Philosophy

TCL's great ideas, without the baggage. Pure metaprogramming with syntax moldable like clay.

"The browser is one example of an inspectable, scriptable runtime being successful. Feather brings that same power to your job runner, web server, database, or desktop app."
